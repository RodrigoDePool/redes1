    REDES DE COMUNICACIONES I

    Comandos útiles para programación

    Volver a: Práctica 1, ses...

    El objetivo principal de este documento es que los alumnos recuerden y fijen conceptos sencillos de programación necesarios para la realización de las prácticas.

    También se trata el problema de la ordenación de los bytes para campos que abarcan más de un byte, problema conocido como endianness.

    Operaciones con cadenas de caracteres

    En el lenguaje de programación C las cadenas de caracteres que contienen secuencias de texto se representan como un array de caracteres. Para declarar una variable de este tipo se hace uso de la siguiente instrucción:
     
    char nombre_variable[longitud_de_la_cadena];
     
    Por ejemplo si queremos definir una cadena de 5 caracteres utilizaremos la siguiente instrucción:
     
    char cadena[5];
     
    Las cadenas pueden ser inicializadas en el momento de declararlas. Por ejemplo, la siguiente sentencia es correcta:
     
    char cadena[5]="Hola";
     
    Se debe tener en cuenta que en C todas las cadenas terminan con el carácter \0 y debe reservarse espacio para el mismo. Es por eso que que si queremos usar una cadena de longitud N debemos definir una variable de N+1 caracteres para almacenarla.
     
    A continuación veremos algunas funciones útiles para la manipulación de cadenas:
     
    • strlen
    Haciendo uso de esta función podemos obtener la longitud de una cadena de caracteres sin tomar en cuenta el carácter \0. Siguiendo el ejemplo anterior podemos definir:


    int longitud;
    char cadena[5]="Hola";
    longitud=strlen(cadena);

    Si imprimimos por pantalla el valor de longitud podemos observar que su valor es 4.


    • strcat
    Esta función permite concatenar 2 cadenas de caracteres para obtener una nueva cadena con el contenido de ambas. La función recibe 2 argumentos que son las cadenas a concatenar. Se debe tener en cuenta que la función concatena las cadenas poniendo el contenido de la variable que se pasa como segundo argumento detrás del contenido de la primera variable. Esto es, en la cadena que se pasa como primer argumento queda almacenada la concatenación de las 2 cadenas. Por lo tanto, la cadena que se pasa como primer argumento debe tener espacio suficiente para contener a ambas cadenas y el carácter de fin de cadena \0. A continuación se muestra un ejemplo de su uso:


    char cadena1[10]="Hola";
    char cadena2[6]="Adios";
    strcat(cadena1,cadena2); 
    Si se imprime por pantalla el contenido de cadena1 después de ejecutar strcat obtendremos la cadena "HolaAdios".

    Existen variantes de esta función que solo concatenan un número determinado de caracteres. Para más información se debe mirar la página de man correspondiente a la función strncat.

    • strcpy
    Esta función permite copiar cadenas de caracteres. La función recibe 2 argumentos y copia el contenido del segundo argumento en el primero. La cadena destino, que se pasa como primer argumento, debe estar reservada y tener suficiente espacio para almacenar la cadena que desea copiarse. A continuación se muestra un ejemplo:


    char cadena2[5]="Hola"";
    char cadena1[5];
    strcpy(cadena1,cadena2);
    Si se imprime por pantalla el contenido de cadena1 después de ejecutar strcpy obtendremos la cadena "Hola". 

    Existen variantes de esta función que solo copian un número determinado de caracteres. Para más información se debe mirar la página de man correspondiente a la función strncpy. 


    • sprintf
    Esta función es similar a la función printf con la diferencia de que en vez de imprimir por pantalla imprime en una cadena. Esta función es muy útil cuando queremos construir cadenas de texto que tiene un formato determinado pero cuyo contenido depende de variables. En nuestro caso, cuando estemos tratando con protocolos de texto plano será útil para construir los mensajes que se intercambian entre los 2 extremos de las comunicaciones. A continuación se muestra un ejemplo del uso de esta función:


    char mensaje1[40],mensaje2[40];
    int numero=0;
     
    sprintf(mensaje1,"Mensaje numero %d\n",numero);
    numero++;
    sprintf(mensaje2,"Mensaje numero %d\n",numero);


    Si se imprime por pantalla el contenido de mensaje1 y mensaje2 después de la ejecución de las funciones sprintf se obtendrán las cadenas "Mensaje numero 0" y "Mensaje numero 1". De esta manera podemos construir cadenas de texto cuando el contenido es dinámico. Al igual que en los casos anteriores la cadena destino debe tener espacio suficiente para albergar el texto que tratamos de imprimir.


    • strstr
    Haciendo uso de esta función podemos buscar una cadena de caracteres dentro de otra. Esto es particularmente útil en nuestro caso cuando queremos analizar mensajes que circulan entre los 2 extremos de una conexión de texto plano y queremos buscar alguna palabra o identificador concreto. La función recibe 2 argumentos y busca la cadena que se pasa como segundo argumento dentro de la cadena que se pasa como primer argumento. En caso de que la cadena se encuentre se devuelve un puntero a la posición de la cadena en la que se encuentra el texto. En caso contrario la función devuelve NULL. A continuación se muestran un ejemplo:
     
    char cadena1[20]="Esto es una cadena";
    char cadena_a_buscar[4]="una";
    char *resultado;
     
    resultado=strstr(cadena1,cadena_a_buscar);
    Si imprimimos por pantalla la variable resultado obtendremos la cadena "una cadena". Hay que tener en cuenta que la variable resultado es un puntero que apunta a cadena1. Si modificamos el contenido de resultado estaremos modificando el contenido de cadena1.

    Operaciones con arrays binarios

    El uso de las cadenas de caracteres nos permitirá trabajar con protocolos que usen texto plano. Por otro lado, existen protocolos de comunicaciones que hacen uso de contenido binario en sus mensajes. Para poder leer y escribir este tipo de mensajes no podemos hacer uso de las funciones de manejo de cadenas y debemos utilizar otras. Para trabajar en este tipo de situaciones haremos uso de los siguientes tipos con longitud y signo definidos especificados en el fichero stdint.h:
     
    • uint8_t
    • uint16_t
    • uint32_t

    Estos tipos definen enteros sin signo de 8, 16 y 32 bits respectivamente (1,2 y 4 bytes). Este tipo de dato nos permiten definir enteros del tamaño que necesitamos ahorrando memoria y haciendo que el tamaño sea fijo independientemente del sistema en el que estemos (El tipo de dato int puede variar su tamaño dependiendo de la arquitectura de procesador en que estemos). Cuando queremos construir mensajes pertenecientes a protocolos binarios se construye un array de bytes o trama que se enviará por la red. Estas tramas se definen generalmente de la siguiente manera:

    uint8_t trama[longitud_trama]={0};
     
    La longitud variará dependiendo del mensaje que queramos enviar. Es útil inicializar todos los bytes de la trama a 0 como se muestra en el ejemplo para evitar tener datos aleatorios no deseados dentro de nuestra trama.
     
    • memcpy
    Al igual que en el caso de las cadenas, en muchas situaciones querremos copiar información que tenemos almacenada en variables dentro de nuestra trama. Esta copia se realiza mediante la función memcpy. La función recibe 3 argumentos: la trama donde se copiará la información, la información a copiar y el tamaño de la información a copiar. A continuación veremos un ejemplo de copia de 3 variables de tamaños diferentes dentro de una trama de manera consecutiva:


    uint32_t posicion=0;
    uint8_t trama[7]={0};
    uint8_t contador1=1;
    uint16_t contador2=14;
    uint32_t contador3=42949672;
     
    memcpy(trama+posicion,&contador1,sizeof(uint8_t));
    posicion+=sizeof(uint8_t);
     
    memcpy(trama+posicion,&contador2,sizeof(uint16_t));
    posicion+=sizeof(uint16_t);
     
    memcpy(trama+posicion,&contador3,sizeof(uint32_t));
    posicion+=sizeof(uint32_t);
    En el ejemplo anterior usamos la variable posición como un indicador del número de bytes que hay que saltarse desde el principio de la trama para escribir la información actual. De esta manera evitamos "pisar" los datos que hemos escrito anteriormente. Se debe tener en cuenta que siempre que estemos copiando de una variable que no sea un puntero (como contador1, contador2 y contador3) se debe poner el símbolo & ya que la función memcpy requiere como entrada punteros a datos. Recordad que en C los punteros y los arrays son equivalentes y por eso en el caso de la variable trama no necesitamos usar el caracter &.

    La función sizeof devuelve el tamaño del dato que se pasa como argumento. Es recomendable poner esta función en vez de usar "números mágicos" ya que ciertos tipos de datos pueden cambiar su tamaño dependiendo del sistema. En el ejemplo utilizamos sizeof tanto para indicar a la función memcpy cuantos bytes deben copiarse como para incrementar la posición de escritura. 

    En el ejemplo anterior se ha mostrado el método para construir una trama. En muchas ocasiones recibiremos una trama de la red y necesitaremos extraer sus campos a variables auxiliares para realizar algún tipo de comprobación o procesamiento. Para ello invertiremos el proceso. En el siguiente ejemplo se muestra como obtener los contadores a partir del contenido de la trama.

    memcpy(&contador1,trama+posicion,sizeof(uint8_t));
    posicion+=sizeof(uint8_t);
    memcpy(&contador2,trama+posicion,,sizeof(uint16_t));
    posicion+=sizeof(uint16_t);
    memcpy(&contador3,trama+posicion,sizeof(uint32_t));
    posicion+=sizeof(uint32_t);
    De esta manera en las variables contador quedará almacenado el valor correspondiente que se encontraba en la trama.

    Operaciones a nivel de bit

    En el lenguaje C la unidad mínima de información que se puede declarar de manera estándar es un byte (en nuestro caso una variable uint8_t). En muchas ocasiones los protocolos que analizaremos o construiremos usarán campos a nivel de bit. Para ello debemos saber manejar este tipo de situaciones. En C existen los siguientes operadores binarios:
     
    • | (or lógico)
    • & (and lógico)
    • << (desplazamiento a la izquierda)
    • >> (desplazamiento a la derecha)
    • ^ (xor lógico)
    • ~ (Complemento a 1)
    Haciendo uso de estos operadores podemos extraer o construir campos a nivel de bit. A continuación se muestra un ejemplo de cómo poner a 1 el bit 12 de una variable de 16 bits:

    uint16_t dato=0x01B0;
    uint8_t uno=1;
     
    dato=dato|(uno << 12);
     
    Si imprimimos la variable dato en hexadecimal obtendremos el valor 0x11B0. Para obtener esto hemos hecho un desplazamiento a la izquierda del número uno en 12 posiciones de tal manera que hemos obtenido una variable que tiene a 0 todos los bits menos el bit 12. Luego se ha realizado un or lógico con la variable que deseamos modificar. De esta manera si en el bit 12 de la variable dato había un 1 quedará un 1 y si había un 0 quedará igualmente un 1.
     
    Con este ejemplo hemos visto como cambiar el valor de un bit. Si lo que queremos es saber el valor de un bit determinado realizaremos un proceso similar. En el siguiente ejemplo se muestra como sacar el valor del bit 12 de una variable.
     
    uint16_t dato=0x11B0;
    uint8_t valor=0;
     
    valor=(dato&0x1000)>>12;
     
    Si imprimimos la variable valor obtendremos el valor 1 ya que el bit 12 de la variable dato es un 1.Para obtener este valor hemos realizado un and lógico de la variable de la que queremos extraer el valor con una máscara. Dicha máscara es un número de 16 bits que tiene todos sus bits a 0 menos el correspondiente a la posición 12 (que es la que queremos extraer). De esta manera, después de realizar la operación and tendremos el valor del bit 12 almacenado en una variable de 16 bits. Como nos interesa tener un único bit, (0 ó 1), debemos mover hacia la derecha con el operador >> 12 posiciones.

    Orden de red y de host

    A lo largo de los años se han desarrollado diferentes tipos de procesadores con diferentes tecnologías. Cada procesador definió un orden a la hora de almacenar los datos de más de 1 byte en memoria. Esta representación de los datos se llama comúnmente "endianness". Existen 3 endianness diferentes Big-endian, Little-endian y middle-endian (bi-endian). En los sistemas Big-endian como lo microprocesadores de Motorola el valor hexadecimal0xDEADBEEF se codificaría en memoria como {DE, AD,BE, EF} mientras que en los sistemas Little-endian como los procesadores Intel y AMD se codificaría como {EF, BE, AD, DE}. OJO hablamos siempre de representación en memoria y no de salida textual. Es decir, en ambas arquitecturas si hacemos un printf de la variable obtendremos el mismo valor. Sin embargo, si realizamos copias a tramas con la función memcpy la variable se copiará de manera diferente en una y otra arquitectura. Como las redes de comunicaciones interconectan diferentes tipos de dispositivos y tecnologías se definió un orden determinado para los datos que circulan por la red. Este orden es Big-endian. Como muchas de las arquitecturas que utilizamos hoy en día son Little-endian debemos realizar un cambio entre orden de red y de host antes de enviar y después de recibir datos de la red. Para realizar esta operación debemos utilizar las funciones:
     
    • htons
    • htonl
    • ntohs
    • ntohl
    Las funciones htons y htonl permiten pasar a orden de red enteros de 16 y 32 bits respectivamente. Las funciones ntohs y ntohl permiten pasar a orden de host enteros de 16 y 32 bits respectivamente. Estas funciones deben ser aplicadas antes de copiar a una trama y después de leer de una trama. A continuación se muestran los ejemplos anteriores de memcpy con las funciones de cambio de orden añadidas.
     
    uint32_t posicion=0;
    uint8_t trama[7]={0};
    uint8_t contador1=1;
    uint16_t contador2=14;
    uint32_t contador3=42949672;
    uint32_t aux32=0;
    uint16_t aux16=0;
     
    memcpy(trama+posicion,&contador1,sizeof(uint8_t));
    posicion+=sizeof(uint8_t);
     
    aux16=htons(contador2);
    memcpy(trama+posicion,&aux16,sizeof(uint16_t));
    posicion+=sizeof(uint16_t);
     
    aux32=htonl(contador3);
    memcpy(trama+posicion,&aux32,sizeof(uint32_t));
    posicion+=sizeof(uint32_t);
     
    uint32_t aux32=0;
    uint16_t aux16=0;
     
    memcpy(&contador1,trama+posicion,sizeof(uint8_t));
    posicion+=sizeof(uint8_t);
     
    memcpy(&aux16,trama+posicion,,sizeof(uint16_t));
    posicion+=sizeof(uint16_t);
    contador2=ntohs(aux16);
    memcpy(&aux32,trama+posicion,sizeof(uint32_t));
    posicion+=sizeof(uint32_t);
    contador3=ntohl(aux32);
     

    Otras funciones útiles

     
    • popen
    Esta función ejecuta un comando de shell desde un programa escrito en lenguaje C. La función recibe 2 argumentos. El primero consiste en una cadena de texto que contiene la orden a ejecutar. El segundo argumento debe ser o bien la cadena "r" para lectura o bien la cadena "w" para escritura.

     
    Si el comando que queremos ejecutar muestra algo por pantalla como salida y queremos que sea accesible desde el programa en C debemos utilizar el modo de lectura ("r"). Por el contrario si el programa que vamos a ejecutar necesita de una entrada por consola y queremos que esa entrada se proporcione desde el programa en C, utilizaremos el modo de escritura "w".


    Si el comando que se ha especificado se puede ejecutar correctamente, popen devuelve un FILE * que a efectos prácticos será como un fichero de texto estándar sobre el que podemos hacer fgets, fwrite, etc.


    A continuación se muestran 2 ejemplos del uso de popen:


    char comando[8]="ls -lah";
    char cadena[255];
    FILE *f;
     
    f=popen(comando,"r");
    if(f==NULL)
    {
    printf("El comando no se ha podido ejecutar\n");
    return -1;
    }
     
    while(fgets(cadena,255,f)!=NULL)
    {
    printf("%s\n",cadena);
    }
    pclose(f);
    En el ejemplo anterior se prepara una cadena con el comando de shell que queremos ejecutar (ls -lah) y se llama a la función popen con el argumento "r" para indicar que queremos leer la salida del comando ls. Si el FILE* que devuelve es NULL, entonces el comando no se ha podido ejecutar. Si no es NULL, el comando se ha ejecutado y en la variable f tenemos un FILE * válido que podemos usar para leer la salida que ha producido el comando ls. Haciendo uso de una instrucción fgets estándar de lectura de fichero podemos leer y mostrar por pantalla desde el programa C el resultado del comando ls ejecutado. Al igual que cuando se abre un fichero hay que cerrarlo antes de terminar el programa, cuando se usa popen hay que usa la función pclose.

    int contador;
    FILE *f;
    char *cadenas[4] = { "5", "3", "1","13"};
    char comando[8]="sort -n";
    f = popen(comando, "w");
    if(f == NULL)
    {
    perror("Error ejecutando el comando");
    return -1;
    }
     
    for(contador=0; contador<4; contador++)
    {
    fputs(cadenas[contador],f);
    fputc('\n',f);
    }
     
    pclose(f);
     
    return 0;
    En el ejemplo anterior se usa el comando sort para ordenar una serie de datos que se han definido en un array de cadenas de caracteres. Para ello se crea una cadena que contiene el comando a ejecutar (sort -n) y se ejecuta la función popen en modo escritura "w". Si el FILE* que devuelve es NULL, entonces el comando no se ha podido ejecutar. Si no es NULL, el comando se ha ejecutado y en la variable f tenemos un FILE * válido que podemos usar para escribir datos que serán pasados al comando sort. Haciendo uso de las instrucciones estándar fputs y fputc podemos pasar la información al comando sort. Al igual que en el caso anterior se debe ejecutar pclose antes de salir del programa.

    Casting sobre punteros

    A veces puede ser útil modificar el tipo de datos del puntero de un array de datos (típicamente conteniendo los datos de un paquete).

    Por ejemplo, supongamos que tenemos un puntero a uint8_t:

    uint8_t * paquete;
    Si queremos coger un campo que ocupa 2 bytes, será necesario hacer el siguiente casting, teniendo igualmente en cuenta las cuestiones de endianness comentadas más arriba :

    uint16_t campo_2_bytes= htons(*(uint16_t *)paquete);
    Igualmente se haría para un campo de 4 bytes:

    uint32_t campo_4_bytes= htonl(*(uint32_t *)paquete);
    Nótese que se hace un cast convirtiendo el puntero a entero de 8 bits a un puntero a entero de 16 o 32 bits según cada caso, y además se obtiene el valor de 16 o 32 bits al que apunta (de ahí el doble asterisco).
